# 股海人生 - MVP 版本

## 怎麼跑起來

```bash
# 安裝依賴 (你應該已經有了)
pip install streamlit pandas

# 進入資料夾
cd stock_life_game

# 啟動遊戲
streamlit run app.py
```

## 檔案說明

| 檔案 | 功能 |
|---|---|
| `app.py` | Streamlit 主程式 + 畫面 |
| `game_core.py` | 玩家狀態、交易、強制平倉、結局判定 |
| `events.py` | 25 個隨機事件 + 觸發邏輯 |
| `market.py` | GBM 模擬股市 (5 檔股票) |
| `jobs.py` | 職業定義 (目前 1 個:工程師) |

## MVP 包含的內容

- ✅ 5 年 60 個月的回合制遊戲
- ✅ 5 檔模擬股票 (2330/2317/0050/2603/6531)
- ✅ 真實股市波動 (GBM 隨機過程)
- ✅ 大牛/大熊/盤整三種市場情緒
- ✅ 25 個隨機事件 (意外/感情/家庭/職場/幸運/陷阱/成長)
- ✅ 強制賣股機制 (現金不足時自動觸發)
- ✅ 4 維狀態 (健康/心情/技能/人脈)
- ✅ 旗標系統 (結婚/有小孩/有房/失業)
- ✅ 6 級結局判定 (S/A/B/C/D/F)
- ✅ 配息 (每年 7 月)
- ✅ 加薪 (每年 +4%)
- ✅ 升職/裁員/副業機會

## 設計筆記

### 為什麼用 GBM 模擬股價?
真實的股票走勢符合「幾何布朗運動」分布——大部分時候小幅波動,偶爾大漲大跌。
這比「每天隨機 ±N%」更真實,讓玩家的決策有意義。

### 為什麼是 5 年 60 個月?
- 太短 (1 年): 事件來不及發生,沒有起伏感
- 太長 (20 年): 一場玩 2 小時,測試成本高
- 5 年: 一場 20-30 分鐘,可以體驗結婚、生子、升職、危機等
  完整人生片段,適合 MVP 驗證

### 為什麼強制賣股優先賣賺最多的?
真實心理:虧損股票會「再等等」,賺錢股票「先鎖利」。
但這個邏輯可以調整 — 你可以改成賣虧最多的 (停損),或按持股比例。

## 你可以怎麼測試「好不好玩」

1. **第一次玩**: 不要看程式碼,純體驗。記下哪裡無聊、哪裡有趣
2. **故意當壞玩家**: All in 一檔股票,看會不會破產
3. **故意當保守玩家**: 都存現金不投資,看結局如何
4. **重複玩 5-10 次**: 感受重玩性,事件是否會重複出現太頻繁

## 已知可以改進的地方

- 事件文字偏單調,缺少「角色性格」
- 沒有圖片,純文字玩久會無聊 (這是 MVP 必要妥協)
- 數值平衡可能太鬆/太緊 (需要實測)
- 5 檔股票太少,可以擴充到 10 檔
